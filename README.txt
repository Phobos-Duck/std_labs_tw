Ипсользуемая бд: Mysql
Таблица бд состоит из столбцов: id, name,current_cmd, set_city, set_category
Бд запускалась на локальном сервере с помощью Open Server panel со стандартными настройками, которые задаются по умолчанию.


Файл token.env состоит из:
NF_TOKEN - токен телеграмм-бота(в моем случает бот находится по адресу https://t.me/Nafanya_AssisnantBot)
WT_TOKEN - api погоды
NW_TOKEN - api нововстей
BD_USER - имя пользователя(для входа в бд)
BD_SERVER - ip сервера бд
BD_NAME - имя бд
BD_TABLE - имя таблицы
BD_PASS - пароль для входа в бд(в моём случае я его не задавала)


По структуре проекта:
main - запуск бота
bot_main - отслеживание команд и основных функций бота
bot_command - команды для взаимодействия с другими файлами и сортировка данных, полученных с api
Config_env - функция получения данных из env файла
config_bd - подключение к бд и функции заполнения/отслеживания
keyboards - файл с клавиатурами
models - определение таблицы
states - категории состояния(какая команда, выбор города, даты, категории новостей и параметры настраиваемые в settings). К сожалению у меня не получилось сделать их с помощью state(), т.к. по какой-то причине не работал set_state у telebot
updates_files - считывание txt файлов для сообщений
api_joke, api_news, api_weather - парсинг сайтов для получения данных. В api_joke пришлось использовать bs4, т.к. открытых api с шутками на русском языке я не нашла.
